Members Only Message Board (Odin Project)

Tech Stack:

  • Front-End: EJS
  • Styling: Bootstrap and vanilla CSS
  • Back-End: Node.js, Express, and PostgreSQL
  • Authentication: Passport and BCrypt

Description:

A secure message board with user authentication implemented using Passport. Users can log in with verified identities, and their sessions remain active until the page is reloaded. For security, passwords are hashed before storage. Access to post authors and the ability to create posts is restricted—users must enter a secret member password to unlock these features. All user data and posts are stored in a PostgreSQL database, ensuring efficient data management and retrieval.

Inventory Managment (Odin Project)

Tech Stack:

  • Front-End: EJS
  • Styling: Bootstrap and vanilla CSS
  • Back-End: Node.js, Express, and PostgreSQL

Description:

This project demonstrates inventory management through Pokémon trainers and their Pokémon parties. Users can create, view, update, and delete individual trainers, as well as manage the Pokémon in their parties. Pokémon information is fetched from the PokeAPI and stored in a PostgreSQL database, ensuring that all details about each Pokémon are readily available. Trainers and Pokémon are maintained in separate tables, allowing for structured data organization and efficient retrieval. This system provides a practical example of relational database management while integrating external API data.
